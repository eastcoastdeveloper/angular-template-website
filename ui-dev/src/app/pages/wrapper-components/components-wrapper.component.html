UI Components
I use components as often as possible. I use web components for native JS applications and Angular components for single page apps. I enjoy creating libraries of components. Passing data between siblings, parents/children, and via services; components are the building blocks of most if not all frameworks.

Using components, sharing data and services, coupled with modules, directives, and fancy sass solutions enable me to develop highly scalable, fast, and mobile friendly apps. This section contains an array of components that can easily be plugged into data streams.

Moreover, I'm a big advocate of modularizing features/pages. This practice enables Angular's lazy loading. Grouping components, importing them to modules, and using observables are the backbone and frontend ingredients of viable SPAs.
