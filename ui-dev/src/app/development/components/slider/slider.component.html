Here's a fun little Angular slider component. I see these often. On a variety of websites. Sometimes called a carousel. There's so many variations of how this can work... so this includes the basics. Spike it a bit to make it your own style. Use an API for the graphics and data. Add smooth transitions or image descriptions. However you decide you modify it, enjoy the feature.

  • Let's begin with an interface of the data for our Angular slider!
  • This is a mirror of what the data looks like. Fortunately for us, the object is rudimentary. More complex objects result in more complex interfaces. Though there are tools that convert data to interfaces, I prefer writing them myself. Practice, practice, practice! 
{{interface}}
  • result is our main array with the afforementioned interface reference.
  • currentIndex is our reference to the currently active book.
  • Set the private _http variable and be sure to add HttpClientModule to the parent module.
  • GET the response and store it in the results variable. There's much more that can be added to this request. Headers, error handling, etc. For the sake of a basic demo, we're not including any of that.
  • changeShowcase resets all the objects' status' values to false. Then sets the incoming value to true.
  • next and prevBook simply retrieve the current index of the original data stream, and reset the status values. We either add via currentIndex++, subtract with currentIndex--, or set it to zero if we reach the end.

  • All very basic Angular here. The main thing to take note of is the ngIf attribute (item.status). When we loop over the data stream, we're saying, only show images that have a status of true. The dots at the bottom are similar in that we use item.status.
