This OMDB API is an example of a data driven UI. Enter a movie or TV show (must be exact) and watch the results populate the UI. By means of OMDB's endpoint, we're fetching data, and populating it accordingly. Though this could no doubt be accomplished in React, Vue, or in native HTML and JavaScript, this sample's in Angular. If you're familiar with TypeScript, it should be no problem. Populate the UI with some content. Then toggle between poster and table layout to get the full scope of the interface.

There's no way around it. If you're creating a UI like this without a library, it's gonna be complex. Plus, factoring in POST requests at every drop like my project called for... Though many would advocate using a library, I try to avoid using libs whenever possible. Especially if it calls for custom solutions. You'll spend more time trying to get the lib to mimic the intended behavior vs creating a system of your own.
